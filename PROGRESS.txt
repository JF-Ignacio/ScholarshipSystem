# First Important Correction

You should NOT duplicate code everywhere.

```php

```

# Proper Structure

Inside:

```txt
config/
```

create:

---

# `admin-auth.php`

```php
<?php

include "session.php";

if (!isset($_SESSION['id'])) {
    header("Location: /TVAM_SCHOLARSHIP/auth/login.php");
    exit();
}

if ($_SESSION['role'] !== 'admin') {
    die("Access Denied.");
}
```

---

# `student-auth.php`

```php
<?php

include "session.php";

if (!isset($_SESSION['id'])) {
    header("Location: /TVAM_SCHOLARSHIP/auth/login.php");
    exit();

    die("Access Denied.");
}
```

---

# THEN


---

Inside student pages:

```php

```

# NOW YOUR NEXT STEP

Now you implement:

# Role-Based Login Redirects

Currently after login:

```php
header("Location: ../admin/dashboard.php");
```

ALWAYS redirects to admin.

That is incorrect for multi-role systems.

---

# Proper Logic

Inside `login.php`

Replace:

```php
header("Location: ../admin/dashboard.php");
```

with:

```php
if ($fetchDB['role'] === 'admin') {
    header("Location: ../admin/dashboard.php");
    exit();
}

if ($fetchDB['role'] === 'student') {
    header("Location: ../student/dashboard.php");
    exit();
}
```

---

# WHY THIS MATTERS

This is called:

# Dynamic Routing Based on Authorization

The system decides:

* where user goes
* based on permissions/roles

Real systems do this everywhere.
